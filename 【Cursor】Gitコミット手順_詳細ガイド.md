# 📖 CursorでGitコミットする詳細手順

## 🎯 ステップバイステップガイド

### ステップ1: ソース管理パネルを開く

#### 方法A: キーボードショートカット
- `Ctrl + Shift + G` を押す

#### 方法B: サイドバーから
1. **左側のサイドバー**を見る
2. 下記のアイコンのいずれかを探してクリック：
   - 📊 **ソース管理** アイコン（フォークのようなアイコン）
   - 🔀 **Git** アイコン
   - 📁 **ファイルエクスプローラー**の下にあるアイコン

#### 方法C: メニューから
1. 上部メニューバーから **「View」** → **「Source Control」** をクリック

---

### ステップ2: ソース管理パネルの見つけ方

開くと、**左側のサイドバー**に以下のような表示が出ます：

```
📁 ソース管理
  ┌─────────────────────────┐
  │ 変更 (8)                │  ← 変更されたファイル数
  │                         │
  │ 📄 ファイル名1          │
  │ 📄 ファイル名2          │
  │ 📄 ファイル名3          │
  │ ...                     │
  └─────────────────────────┘
```

---

### ステップ3: ファイルをステージングする方法

#### 方法A: ファイルごとに「+」ボタンをクリック

各ファイルの右側（または左側）に **「+」ボタン** があります：

```
┌──────────────────────────────────────┐
│ 📄 【完成】接続文字列.env      [+]  │  ← この「+」をクリック
└──────────────────────────────────────┘
```

**「+」ボタンが見つからない場合**：
- ファイル名の上に**マウスをホバー**すると表示される
- または、ファイル名の**右側**をクリックすると表示される

#### 方法B: 右クリックメニュー

1. **ファイル名を右クリック**
2. メニューから **「ステージ」** または **「Stage Changes」** を選択

#### 方法C: すべて一度にステージング

1. **「変更」セクションの右側**に **「+」ボタン**がある
   - 「変更 (8)」の横にある **「+」ボタン**をクリック
   - すべてのファイルが一度にステージングされます

2. または、**「すべてステージ」**というボタンがある場合

---

### ステップ4: ステージングの確認

ファイルをステージングすると、**「ステージング済み」**セクションに移動します：

```
📁 ソース管理
  ┌─────────────────────────┐
  │ ステージング済み (8)    │  ← ここに移動
  │ 📄 ファイル名1          │
  │ 📄 ファイル名2          │
  │ ...                     │
  └─────────────────────────┘
  ┌─────────────────────────┐
  │ 変更 (0)                │  ← ここは空になる
  └─────────────────────────┘
```

---

### ステップ5: コミットメッセージを入力

1. **ステージング済み**セクションの**上**に、**テキストボックス**があります
2. そこにコミットメッセージを入力：

```
┌─────────────────────────────────────┐
│ メッセージ（Ctrl+Enterでコミット） │
│                                     │
│ 🔒 セキュリティ修正: 接続文字列から│
│    実際の認証情報を削除            │
│                                     │
│ - 実際のDB接続文字列（パスワード含 │
│  む）をプレースホルダーに置換      │
│ - .gitignoreに*.envパターンを追加  │
│ - GitHubの警告に対応               │
└─────────────────────────────────────┘
```

---

### ステップ6: コミットを実行

#### 方法A: キーボードショートカット
- `Ctrl + Enter` を押す

#### 方法B: ボタンをクリック
- コミットメッセージボックスの下にある **「✓ コミット」** ボタンをクリック

---

### ステップ7: プッシュ

1. コミット後、**上部に通知**または**ボタン**が表示されます
2. **「プッシュ」** または **「Push」** ボタンをクリック
3. または、**「...」メニュー**から **「プッシュ」** を選択

---

## 🔍 「+」ボタンが見つからない場合

### 確認ポイント

1. **「変更」セクションが表示されているか確認**
   - ファイルが変更されていない場合、何も表示されません

2. **ファイル名の上にマウスを置く**
   - ホバーするとボタンが表示されることがあります

3. **右クリックメニューを試す**
   - ファイル名を右クリック → 「ステージ」を選択

4. **「すべてステージ」ボタンを探す**
   - 「変更」セクションの上や下にボタンがある場合があります

## 🆘 それでも見つからない場合

### 代替方法：ターミナルからGitコマンドを実行

Cursorの統合機能が使えない場合、ターミナルから直接実行することもできます：

1. **ターミナルを開く**
   - `Ctrl + Shift + `` （バッククォート）
   - または、メニューから「Terminal」→「New Terminal」

2. **以下のコマンドを実行**：

```bash
# すべての変更をステージング
git add .

# コミット
git commit -m "🔒 セキュリティ修正: 接続文字列から実際の認証情報を削除

- 実際のDB接続文字列（パスワード含む）をプレースホルダーに置換
- .gitignoreに*.envパターンを追加して.envファイルを保護
- GitHubの警告に対応
- TiDB Cloudのパスワードをリセットして無効化
- Renderの環境変数を新しいパスワードに更新"

# プッシュ
git push
```

---

## 📝 まとめ

1. `Ctrl+Shift+G` でソース管理パネルを開く
2. 「変更」セクションにファイルが表示される
3. ファイルの横の「+」ボタンをクリック（または右クリック→「ステージ」）
4. コミットメッセージを入力
5. `Ctrl+Enter` でコミット
6. 「プッシュ」ボタンをクリック

**「+」ボタンが見つからない場合は、右クリックメニューから「ステージ」を選択してください！** ✅
