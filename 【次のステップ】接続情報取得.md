## 🔗 ステップ5: TiDB Cloudで接続情報を取得

### 1. cluster0 を開く

1. TiDB Cloudダッシュボードで **cluster0** をクリック
2. クラスターの詳細ページを開く

### 2. Connectタブを開く

1. 上部のタブから **「Connect」** をクリック
2. 接続方法が表示されます

### 3. 接続文字列を取得

1. **「Standard Connection」** または **「General」** を選択
2. **接続文字列をコピー**

**接続文字列の形式:**
```
mysql://username:password@tidb.xxxxx.aws.tidbcloud.com:4000/database?ssl-mode=REQUIRED
```

または、個別に表示される場合:
- **Host**: `tidb.xxxxx.aws.tidbcloud.com`
- **Port**: `4000`
- **User**: `username`
- **Password**: `password`（設定したもの）
- **Database**: `space_mood_oracle`

### 4. パスワードを設定（初回のみ）

- 初回接続時はパスワードを設定する必要があります
- パスワードを安全に保管してください

### 5. 接続文字列を組み立てる

個別の情報が表示される場合、以下の形式で組み立てます:

```
mysql://[User]:[Password]@[Host]:[Port]/[Database]?ssl-mode=REQUIRED
```

**例:**
```
mysql://root:yourpassword@tidb.abc123.aws.tidbcloud.com:4000/space_mood_oracle?ssl-mode=REQUIRED
```

## ⚙️ ステップ6: 環境変数に設定

### Render用（後で使用）

接続文字列をコピーして、後でRenderの環境変数として設定します:

```env
DATABASE_URL=mysql://username:password@tidb.xxxxx.aws.tidbcloud.com:4000/space_mood_oracle?ssl-mode=REQUIRED
JWT_SECRET=your-random-secret-key-change-this-in-production
NODE_ENV=production
PORT=10000
```

### ローカル用（オプション）

ローカルでテストする場合、`.env` ファイルを作成:

```env
DATABASE_URL=mysql://username:password@tidb.xxxxx.aws.tidbcloud.com:4000/space_mood_oracle?ssl-mode=REQUIRED
JWT_SECRET=your-random-secret-key-change-this-in-production
```
