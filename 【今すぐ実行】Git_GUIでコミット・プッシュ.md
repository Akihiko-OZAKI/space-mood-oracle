# 🚀 Git GUIでコミット・プッシュする手順

## 📋 現在の状況

Git GUIが立ち上がっています！**まず、リポジトリを開く必要があります。**

---

## 🎯 ステップバイステップ

### ステップ1: 既存のリポジトリを開く ⭐重要⭐

1. **「Open Existing Repository」をクリック**
   - 画面に3つのリンクが表示されています
   - その中の「**Open Existing Repository**」（既存のリポジトリを開く）をクリック

2. **フォルダを選択**
   - フォルダ選択ダイアログが開きます
   - 以下のフォルダを選択：
   ```
   H:\AI_study\228_宇宙パワー_V1\space_mood_oracle_v2.1\space_mood_oracle
   ```
   - 「選択」または「OK」をクリック

3. **Git GUIのメイン画面が開く**
   - リポジトリが開かれると、変更されたファイルが表示される画面になります

---

### ステップ2: 変更を確認（Rescan）

1. **Git GUIの画面で「Rescan」ボタンをクリック**
   - 画面の上部または中央あたりに「Rescan」ボタンがあります
   - これで変更されたファイルが読み込まれます

### ステップ3: 変更されたファイルを確認

1. **「Unstaged Changes」セクションを確認**
   - 左側または中央に変更されたファイルのリストが表示されます
   - 8つのファイルが表示されるはずです

2. **ファイルを選択**
   - コミットしたいファイルをクリックして選択
   - **すべてのファイルをコミットする場合**：ファイルを1つずつ選択するか、「Stage All」ボタンを探す

### ステップ4: ファイルをステージング

#### 方法A: 各ファイルを個別にステージング

1. **ファイルを選択**
   - 「Unstaged Changes」にあるファイルをクリック

2. **「Stage Changed」ボタンをクリック**
   - ファイルの右側、または画面下部に「Stage Changed」ボタンがあります
   - クリックするとファイルが「Staged Changes」に移動します

3. **すべてのファイルをステージング**
   - 各ファイルについて上記を繰り返す
   - または、「Stage All」ボタンがある場合はそれをクリック

#### 方法B: すべて一度にステージング

1. **「Stage All」ボタンを探す**
   - 画面の上部やボタンエリアに「Stage All」がある場合、それをクリック
   - すべてのファイルが一度にステージングされます

### ステップ5: コミットメッセージを入力

1. **「Staged Changes」セクションを確認**
   - ステージングしたファイルが表示されていることを確認

2. **コミットメッセージを入力**
   - 画面下部にテキストボックスがあります
   - 以下のメッセージをコピー＆ペースト：

```
🔒 セキュリティ修正: 接続文字列から実際の認証情報を削除

- 実際のDB接続文字列（パスワード含む）をプレースホルダーに置換
- .gitignoreに*.envパターンを追加して.envファイルを保護
- GitHubの警告に対応
- TiDB Cloudのパスワードをリセットして無効化
- Renderの環境変数を新しいパスワードに更新
```

### ステップ6: コミットを実行

1. **「Commit」ボタンをクリック**
   - コミットメッセージの下、または画面下部に「Commit」ボタンがあります
   - クリックするとコミットが完了します

### ステップ7: プッシュ

1. **「Push」ボタンを探す**
   - 画面の上部メニューまたはボタンエリアに「Push」ボタンがあります

2. **「Push」をクリック**
   - 確認ダイアログが表示される場合があります
   - 「OK」または「Push」をクリック

3. **認証が必要な場合**
   - GitHubのユーザー名とパスワード（またはトークン）の入力が求められる場合があります
   - 入力して「OK」をクリック

## 🆘 よくある問題と解決方法

### 「Stage Changed」ボタンが見つからない

- **ファイルを選択**しているか確認
- ファイルをダブルクリックしてみる
- 「Stage All」ボタンがある場合はそれを試す

### 「Push」ボタンが見つからない

- 上部メニューの「Remote」→「Push」を探す
- または、「Repository」→「Push」

### 認証エラーが出る

- GitHubのユーザー名とパスワードを確認
- パスワードではなく**Personal Access Token**が必要な場合があります
  - GitHub → Settings → Developer settings → Personal access tokens → Generate new token

### 「Rescan」を押しても何も表示されない

- ファイルが既にコミットされている可能性があります
- または、Gitリポジトリが正しく初期化されていない可能性があります
