# TiDB Cloud確認: testデータベースのテーブル確認手順

## 🎯 確認目的

`test` データベースに必要なテーブルが存在するか確認します。

---

## 📋 確認手順

### ステップ1: TiDB Cloudコンソールを開く

1. https://tidbcloud.com にアクセス
2. ログイン
3. クラスターを選択

---

### ステップ2: SQL Editorを開く

1. 左サイドバーの **「SQL Editor」** をクリック
2. SQL Editorが開く

---

### ステップ3: testデータベースに接続

以下のSQLを実行：

```sql
USE test;
```

---

### ステップ4: テーブルの一覧を確認

以下のSQLを実行：

```sql
SHOW TABLES;
```

---

### ステップ5: 必要なテーブルを確認

以下のテーブルが表示されるべきです：

- `daily_sentiment_scores`
- `predictions`
- `space_weather_data`
- `users`

---

## 🔍 確認結果による対応

### ケース1: テーブルが存在しない

**症状:**
- `SHOW TABLES;` の結果が空
- または、必要なテーブルが一部しかない

**解決方法:**
- テーブルを作成する必要があります
- 次のステップでテーブル作成のSQLを提供します

---

### ケース2: テーブルが存在する

**症状:**
- 必要なテーブルがすべて表示される

**次のステップ:**
- データが存在するか確認
- もしデータが存在しない場合、それは正常な状態です
- `/lab` ページから「実データ取得（NOAA）」を実行してデータを投入

---

### ケース3: 一部のテーブルだけ存在する

**症状:**
- 一部のテーブルは存在するが、一部は存在しない

**解決方法:**
- 存在しないテーブルを作成する必要があります

---

## 📝 次のステップ

### テーブルが存在しない場合

1. テーブル作成のSQLを実行
2. 再度 `SHOW TABLES;` で確認
3. 公開UIを再読み込みしてエラーが解消されたか確認

### テーブルが存在する場合

1. データが存在するか確認：
   ```sql
   SELECT COUNT(*) FROM daily_sentiment_scores;
   SELECT COUNT(*) FROM predictions;
   SELECT COUNT(*) FROM space_weather_data;
   ```
2. データが存在しない場合、それは正常です（エラーではありません）
3. `/lab` ページから「実データ取得（NOAA）」を実行

---

**まず、TiDB CloudのSQL Editorで `USE test; SHOW TABLES;` を実行して、テーブルが存在するか確認してください！** 🔍
